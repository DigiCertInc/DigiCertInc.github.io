修改SQLServer2005默认端口

扫1433的黑阔实在太多了，所以为了提高服务器的安全性建议修改掉服务器SQLServer的默认端口1433，2005修改方法是在SQL Server Configuration Manager里面（菜单位置如图）

在SQLServer 2005网络配置-》MSSQLServer的协议-》TCP/IP里面修改掉《IP地址》里面的默认1433端口，如修改成3413后的效果图

然后重启下MSSQLServer服务就是修改成3413端口了

按照以上设置完成客户端端口更改，在程序中的数据库连接字符串也应该做相应的更改，否则还是无法连接打数据库。
Provider=SQLNCLI.1;Persist Security Info=True;User ID=**;Password=;Initial Catalog=tablename;Data Source=88.88.88.88,3413.
